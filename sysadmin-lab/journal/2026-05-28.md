# 2026-05-28 - Linux Essentials: permisos

## Practica

Permisos Linux aplicados a los scripts del laboratorio IntraCorp Helpdesk.

## Objetivo

Aprender a leer permisos `rwx`, entender propietarios y grupos, y ver como Git guarda el bit ejecutable de scripts.

## Hallazgo

Desde WSL en una ruta montada de Windows (`/mnt/c/...`), `ls -lah scripts` puede mostrar permisos como:

```text
rwxrwxrwx
```

Pero Git puede tener esos mismos scripts guardados como:

```text
100644
```

Eso significa que Git no los considera ejecutables, aunque el filesystem montado los muestre con `x`.

## Accion aplicada

Se documenta la practica:

```text
sysadmin-lab/challenges/12-linux-permisos-ejecutables.md
```

Tambien se marcan los scripts `.sh` como ejecutables en Git para que al clonar el repo en Linux queden como `100755`.

## Comandos importantes

```bash
ls -lah scripts
git ls-files --stage scripts/*.sh
git update-index --chmod=+x scripts/*.sh
git diff --summary
```

## Aprendizajes

- `r` significa lectura.
- `w` significa escritura.
- `x` significa ejecucion.
- `100644` es archivo normal en Git.
- `100755` es archivo ejecutable en Git.
- En WSL sobre `/mnt/c`, lo que muestra el filesystem puede no ser igual a lo que Git versiona.
- Para scripts del laboratorio, el bit ejecutable mejora la experiencia al correrlos como `./scripts/script.sh`.

## Nota personal

Hoy el avance fue pequeno pero practico: entender permisos desde un caso real del repo, no desde ejemplos aislados.
